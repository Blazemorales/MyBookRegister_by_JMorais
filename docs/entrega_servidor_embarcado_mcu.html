Objetivo

Demonstrar um servidor embarcado em MCU (ESP32) capaz de expor, publicamente pela internet e via HTTP/HTTPS, o estado real de um atuador (lâmpada DC 12V acionada por relé) e de receber um sinal de ação — enviado manualmente pelo operador a partir da interpretação do estado exibido remotamente. O acesso público não depende de plataformas de terceiros (Arduino Cloud ou similar): o servidor web roda inteiramente no próprio ESP32.

Arquitetura

navegador (qualquer lugar, ex.: 4G) | HTTPS publico v Cloudflare Tunnel <-- cloudflared rodando na Raspberry Pi | HTTP (rede local) v ESP32 -- servidor web embutido (WebServer.h), sem dependencia de nuvem de terceiros | GPIO4 (digitalWrite/digitalRead) v Modulo rele 5V (SRD-05VDC-SL-C) --contatos COM/NO--> Lampada DC 12V

A ESP32 conecta automaticamente na rede disponível — UnB Wireless (WPA2-Enterprise/PEAP-MSCHAPv2) ou rede residencial (WPA2-Personal) — escaneando as redes visíveis a cada boot/reconexão, sem precisar reprogramar o firmware ao trocar de local.

Como os requisitos do trabalho são atendidos

RequisitoImplementação
Acesso remoto via HTTP/HTTPS, publicamenteCloudflare Tunnel expõe o servidor da ESP32 num hostname HTTPS público, acessível de qualquer rede (não USB/Bluetooth/WiFi Direct)
Leitura de dado real, sem simulaçãoO estado exibido (ligada/desligada) é lido diretamente do pino físico do relé via digitalRead(), refletindo o acionamento real do circuito
Sinal de ação por interpretação da leituraAção manual: o operador acompanha o estado remotamente e decide ligar/desligar pelos botões da página web
Sem plataforma de terceiros (Arduino Cloud etc.)Servidor web nativo no ESP32 (WebServer.h); o Cloudflare Tunnel é só infraestrutura de rede, não hospeda lógica do projeto
